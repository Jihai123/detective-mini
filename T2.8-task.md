## 5. endingMatrix 设计

### 评分体系

候选 B:canonical=2 / partial=1 / misread=0 / 不发现=0
- 全部 10 条 clue 都参与评分(不论是否参与 confrontation 攻击)
- 上限 = 20 分

阈值:
- 完美 ≥ 14
- 部分 8-13
- 错误 ≤ 7

关键证据(诡计承重柱,完美结局触发条件):
- c003(录音文件+定时任务)
- c006(运营商通话记录)
- clue-wangkai-recording(王凯录音原文件)

### endingMatrix.rules 完整设计

```
rules: [
  // 完美三变体(顺序:具体在前)
  { when: {
      submissionCorrect: true,
      clueInterpretations: { 'clue-hospital-bill': 'canonical' }
    },
    requires: {
      totalScore: '>=14',
      keyCluesAllCanonical: ['c003', 'c006', 'clue-wangkai-recording']
    },
    endingKey: 'perfect-B' },

  { when: {
      submissionCorrect: true,
      clueInterpretations: { 'clue-hospital-bill': 'misread' }
    },
    requires: {
      totalScore: '>=14',
      keyCluesAllCanonical: ['c003', 'c006', 'clue-wangkai-recording']
    },
    endingKey: 'perfect-C' },

  { when: { submissionCorrect: true },
    requires: {
      totalScore: '>=14',
      keyCluesAllCanonical: ['c003', 'c006', 'clue-wangkai-recording']
    },
    endingKey: 'perfect-A' },

  // partial 结局(指认对但条件不达)
  { when: { submissionCorrect: true },
    requires: { totalScore: '>=8' },
    endingKey: 'partial' },

  // 错误变体
  { when: { submissionWrongTarget: 'wangkai' },
    endingKey: 'wrong-A' },
],
fallback: 'wrong-B'
```

注:`requires` 字段与 `when` 一起判定。`requires` 不满足时跳过本规则继续
下一条。`when` 命中但 `requires` 不达时不视为命中。

完整 ending key 清单:
- perfect-A / perfect-B / perfect-C(完美三变体)
- partial(指认对但分数 8-13 或关键证据非全 canonical)
- wrong-A(指认王凯)
- wrong-B(其他错误,fallback)

```typescript
// TestimonySentence 加两个 optional 字段
interface TestimonySentence {
  // ... 现有字段
  breakable?: boolean;       // 默认 true
  unlocksSuspect?: string;   // canonical 攻击时解锁该 suspect tab
}

// ConfrontationState (per suspect) 加 unlocked
interface ConfrontationState {
  // ... 现有字段
  unlocked: boolean;         // 默认 true
}

// EndingMatrixRule.when 扩展
interface EndingMatrixRuleWhen {
  submissionCorrect?: boolean;
  emotionalState?: ...;          // 已有,保留
  clueInterpretations?: { [clueId: string]: ClueInterpretation };  // 新增
  submissionWrongTarget?: string;  // 新增
}

// EndingMatrixRule 加 requires
interface EndingMatrixRule {
  when: EndingMatrixRuleWhen;
  requires?: {
    totalScore?: string;             // 例 '>=14'
    keyCluesAllCanonical?: string[]; // 必须全 canonical 的 clue id 列表
  };
  endingKey: string;
}
```

## 14. unlocksSuspect 机制规范(全新章节)

### 14.1 设计意图

case-002 是"红鲱鱼"叙事:玩家初次接触三个角色(陈昊 / 王凯 / 李婷)时,
王凯**主动交代矛盾**和**偷录死者**的行为最为可疑,玩家本能怀疑王凯。
通过用 c003(录音文件)对质王凯 t2(偷录背景),王凯解释"录音不是我给的,
是有人从死者飞书拿的" — **反转触发**,嫌疑指向陈昊。

T2.6-B 多嫌疑人 confrontation 机制原本设计是"每个 suspect 都有可击破
sentence"。case-002 的设计是"嫌疑人可询问可对峙,只有真凶才需要被
击破"。这与原设计冲突 — T2.8 引入 breakable + unlocksSuspect 字段
解决该冲突。

### 14.2 字段语义

```
TestimonySentence.breakable?: boolean   (默认 true)
  - true:canonical 攻击命中 → 计入 won,roundResult 推进
  - false:canonical 攻击命中 → 不计 won,但触发 unlocksSuspect 副作用
          (若有)+ 渲染 feedback 文本

TestimonySentence.unlocksSuspect?: string
  - 仅当 breakable === false 时生效
  - canonical 攻击命中后,设置 confrontationBySuspect[unlocksSuspect].unlocked = true

ConfrontationState.unlocked: boolean   (默认 true)
  - false:该 suspect tab 在 confrontation UI 中显示为锁定状态(灰显 + 锁图标)
  - true:可点击切换
```

### 14.3 case-002 confrontation 启动状态

- 默认 currentSuspectId = 'wangkai'(王凯 tab 显示高亮)
- chenhao tab 显示 "🔒 嫌疑人未明" 灰显占位
- liting 不出现在 confrontation tab 列表

### 14.4 wangkai-t2 用 c003 canonical 攻击的运行时流程

1. 玩家选中 wangkai-t2,从证据栏选 c003 攻击
2. resolveOutcome 返回 canonical(因 wangkai-t2.responses.byClue.c003 配置 canonical)
3. 检查 sentence.breakable === false → 不计 won
4. 检查 sentence.unlocksSuspect === 'chenhao' → 设置
   confrontationBySuspect['chenhao'].unlocked = true
5. 渲染 feedback 文本(王凯诧异后冷静的反应文本,见 testimony.md 王凯 T2 段)
6. UI 触发 chenhao tab 解锁视觉变化(从灰显切换为可点击)
7. 持久化保存 confrontationBySuspect

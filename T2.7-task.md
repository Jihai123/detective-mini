# T2.7 — case 导入架构(JSON 外置 + 素材命名约定自动加载)

## 0. 上下文

T2.6 + T2.6 hotfix 完成后,case-002 启动机制全部到位,但有一个结构性障碍:
case 数据(suspects / clues / endings / endingMatrix / dialogue / sentences)目前
全部 hardcode 在 TypeScript 文件 `src/cases/case-001/data.ts`。

case-002 数据量是 case-001 的约 3 倍。如果继续 hardcode TS,会出现:
1. 数据 + 代码混在一起,review 时数据修改触发 "TS 改了" 的误读
2. 翻译 / 美工 / 策划要修文案必须懂 TS 才能改
3. 未来 case-003+ 会指数级膨胀
4. T2.8 数据化时 PR diff 会非常巨大,review 不可控

**T2.7 解决方案**: case 数据搬入 JSON,素材按命名约定自动加载,代码侧
零改动即可新增 case。

## 9. 风险点

- JSON schema cast 后类型丢失
- 素材路径约定与现有引用方式冲突 — 盘点必须穷举所有引用点
- 动态加载方案的 dispose 时机 — 需要 abort controller 或 race flag
- case-001 行为不变验收的覆盖度 — 必须逐节点对比 TS 原值与 JSON 加载后值
- ConfrontationConfig 双格式兼容(rounds vs suspects)JSON 化后能否正确 cast

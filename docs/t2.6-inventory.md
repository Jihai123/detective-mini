## 3.2 confrontation runtime 现状

### ConfrontationState 字段清单

- **定义位置**: `src/stage1/types.ts:223-230`

```ts
type ConfrontationState = {
  roundIndex: number;               // 当前 round 索引(0-based)
  mistakesInCurrentRound: number;  // 本 round 错误次数
  roundResults: Array<'pending' | 'won' | 'lost'>;  // 每 round 结果
  selectedSentenceId: string | null;
  status: 'idle' | 'ongoing' | 'success' | 'allLost';
  lastFeedback: string;            // 本次反应文本
}
```

### "对哪个嫌疑人对质"的记录方式

- **单嫌疑人 hardcode 在 case data 顶层**:`caseConfig.confrontation.target: string`
- **case-001**: `target: 'zhoulan'`(固定)
- **无 currentSuspectId 概念**,`ConfrontationState` 里无任何嫌疑人 id 字段
- runtime 读取:`src/stage1/app.ts:877` `caseConfig.characters.find(c => c.id === caseConfig.confrontation.target)`

### attacksTestimonyIds 判定后,反应文本来源

- **来源: case data → ConfrontationRound 字段**
  - 命中:`ConfrontationRound.onCorrectFeedback: string`
  - 未命中:`ConfrontationRound.onWrongFeedback: string`
  - 全败:`ConfrontationConfig.onAllLost?: string`
- **dispatch**: `src/stage1/app.ts:509` 写入 `state.confrontation.lastFeedback`
- **渲染**: `src/stage1/app.ts:944` `<p class="confront-feedback">${conf.lastFeedback}</p>`
- **结论**: 反应文本存在 data 层,无组件模板生成

### "本 round 平/败/胜" 当前分类

- `'won'` = 玩家出示正确证据(counterEvidenceId 命中)
- `'lost'` = 本 round 错误次数 ≥ maxMistakes
- `'pending'` = 尚未完成
- **无"平"分类**(不是 won 就是 lost,没有 draw)
- 最终判定:`src/stage1/app.ts:550` `wonCount > lostCount` 则为 success,否则回到调查

## 3.3 interpret overlay 当前流程

### 三档选项是否等权可选

- **canonical**: 始终存在(必选项)
- **partial**: 可选项,仅当 `clue.interpretations.partial` 存在时显示
- **misread**: 可选项,仅当 `clue.interpretations.misread` 存在时显示
- **case-001**: 4 条 clue 全有 canonical + misread,**无 partial**

### 已选解读能否重选

- **可以重选**,无锁定机制
- `src/stage1/app.ts:717-722`:打开 overlay 时加载已有选择为初始态
- `src/stage1/app.ts:674-685`:setInterpretation 直接覆盖已有条目

### UI 层是否有视觉区分

- **无视觉区分**
- 所有档位按钮使用相同 class `interpret-option`,仅 `is-selected` 区分已选/未选
- 选项顺序通过 clueId hash 打乱(`clueIdHash(id) % 2 === 1` 则 reverse)
- 玩家看不到哪个是 canonical/partial/misread

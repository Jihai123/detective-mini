## 结局文本风格统一性说明 🟡 `[DRAFT]`

所有结局遵循以下原则：

1. **侦探总结**：简短、事实性、不煽情，用于梳理玩家的破案逻辑
2. **陈昊独白**：有情感重量但不过度戏剧化；在完美结局中承认，在部分/错误结局中保留防御
3. **结局陈述**：冷静叙述后续事件，避免"正义得到伸张"式的满足感或"邪不胜正"式的狗血
4. **共同氛围**：压抑、现实、让玩家自己感受道德重量而不是被告知

这种风格**不提供宣泄**，也**不煽情**。符合你在早期定下的"这就是现实中的某种人"的灰度基调。

## 🔴 `[TODO]` 需你决策的结局细节

以下几点建议你在玩测前明确：

1. **陈昊独白的长度上限**：完美-B 版当前最长（约 350 字）。要不要更短？是否需要分段显示（一段一段播放 vs 整段一次性展示）？

2. **错误-A 结局中"陈昊升职接替林泽远"**：这个细节很刺痛，但也很戏剧化。你是否接受？如果觉得过于"巧合"，可以改为"陈昊没有升职，但也没有被裁"——同样达到"真凶得利"的效果但更克制。

3. **"案件结束"作为收尾文案**：每个结局都用这句话结束。是否需要给每个结局单独的 closing line（比如完美结局用"案件结束"，错误结局用"案件归档"等）？

4. **结局文案的 UI 呈现**：文案本身写好了，但最终怎么显示（打字机效果？分段展示？配图？）是产品层面的决定，不在本文档范围。
